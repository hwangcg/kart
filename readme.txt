
[한국예술영재교육연수원 작업 참고문서]


[기본 사항]

- 크롬 브라우저에 최적화 되어 있고, ie10까지 준수하였습니다.

- 모바일은 아이폰6s / V30 기기로 테스트 진행했습니다.

- W3 validator 검증 결과 form 요소에 name이나 id를 부여하지 않은 요소를 제외하고는 Error가 없었습니다.
    -> Warning으로 표시되는 항목들이 있으나, section 태그에 heading 태그가 존재하지 않는다는 정도의 경고 사항이기 때문에 이후 작업 진행하셔도 무방할거라 판단됩니다.

- 모바일 375px까지 반응형으로 제작되었고, 그 이하 320px(아이폰5/se), 280px(갤럭시 폴드) 해상도에서는 375px기준으로 보여지도록 viewport 처리하였습니다.
    -> common.js line:66 참고하시면 됩니다.
    -> 화면이 렌더링 되는 순간 기기와 스크린 사이즈에 따른 뷰포트를 설정하기 때문에 브라우저 상에서 강제로 resizing 할 경우 틀어져 보일 수 있습니다. 
       혹시나 이런 상황이 있으시다면 새로고침 후에 다시 확인해주시면 됩니다.



[개발 참고사항]

- 파일 네이밍은 [한국예술종합학교 학습자 화면설계_v1.1_201130] 기획서를 기준으로 '기획서페이지_영문명.html'로 작성했습니다.

- 팝업은 전달받은 디자인 파일(팝업창2.psd)을 기준으로 99_popup.html에 작업해두었습니다.
    -> 나의강의실 팝업의 경우 작업 범위가 아니었고, 김두만과장님의 별도 요청에 따라 팝업 레이아웃만 잡아두었습니다.

- 몇가지 스크립트 요소를 common.js에 정의해두었기 때문에, 메인화면과 서브화면 모두에서 import 해주셔야 합니다.

- reset.css, common.css는 메인과 서브 페이지 모두에서 필요하며, main.css는 메인페이지에서만, sub.css파일은 서브페이지에서만 import하셔도 무방합니다.

- header와 footer의 경우 main 페이지에는 태그가 들어있고, sub 페이지들의 경우에는 layout폴더에 있는 header.html과 footer.html을 include하는 형태로 작업되어 있습니다.
    -> main.html에 있는 header/footer 태그와 layout폴더에 있는 파일들의 내용은 완전히 같습니다.

- sub 레이아웃은 main과 달리 body 태그 바로 밑에 flex_wrap, flex_con 이라는 클래스를 가진 div태그가 있는 점 주의 부탁드립니다.(컨텐츠 길이 짧아도 footer가 하단에 고정되어 있게 하기 위함)



[외부 라이브러리]

- jquery-3.5.1.min.js가 사용되었습니다.

- 메인페이지 과정 슬라이드를 위해 swiper.min.css, swiper.min.js 가 사용되었습니다.

- bootstrap-grid v4 프레임워크로 전체 layout을 작성했고, 디테일한 작업을 위해 576px, 768px, 992px, 1200px 총 4단계의 media query가 사용되었습니다.
    -> 디테일한 부분들은 프레임워크를 이용하지 않고 직접 작업했습니다.
    -> 현재 import한 파일은 압축되지 않은 bootstrap-grid.css 파일이니 필요하시면 bootstrap-grid.min.css 파일로 변경해주셔도 됩니다.
